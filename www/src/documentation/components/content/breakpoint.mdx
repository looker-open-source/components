The `Breakpoint` component is used to render content at desired screen sizes.

While we recommend using css breakpoints or styled-system when possible, this component is provided to facilitate larger markup or behavior changes tailored to the device.

We support five named breakpoints as follows:

## Rendering content on a range of devices

Use the `show` prop to specify either a string name of a single device (e.g. `'mobile'`), or a 2-length tuple to define a range of devices (`['mobile', 'desktop']` &ndash; "render from mobile to desktop").

```jsx
<Paragraph mb="u3">Resize window to watch content change:</Paragraph>
<Breakpoint show={['mobile', 'laptop']}>
  <Box2 bg="criticalAccent" p="u5">
    Render on Mobile, Tablet, and Laptop (0-1024px)
  </Box2>
</Breakpoint>
<Breakpoint show={['desktop', 'xl']}>
  <Box2 bg="keyAccent" p="u5">
    Desktop and xl screens (1025px and larger)
  </Box2>
</Breakpoint>
<Breakpoint show="desktop">
  <Box2 bg="positiveAccent" p="u5" mt="u3">
    Show on desktop only (1025-1200px)
  </Box2>
</Breakpoint>
<Breakpoint show="xl">
  <Box2 bg="informAccent" p="u5" mt="u3">
    Show on XL only (>1200px)
  </Box2>
</Breakpoint>
```
