The `<MessageBar />` component is used to alert the user with `warn`, `critical`, `inform`, or `positive` messages, settable via the `intent` property. If no intent is set, it will fall back to `inform` by default.

```jsx
<SpaceVertical gap="xsmall">
  <MessageBar intent="inform">Inform (default)</MessageBar>
  <MessageBar intent="warn">Warning</MessageBar>
  <MessageBar intent="positive">Positive</MessageBar>
  <MessageBar intent="critical">Critical</MessageBar>
</SpaceVertical>
```

## canDismiss

If you don't want the MessageBar to be clearable, you can remove the close button by setting `canDismiss` to false.

```jsx
<MessageBar canDismiss={false}>I can never be closed</MessageBar>
```

## Controlled Component Pattern

MessageBar will accept an `onDismiss` callback along with the `visible` boolean prop for use in the controlled component pattern. This can be useful if you wish to toggle display externally or provide an Undo option.

```jsx
;() => {
  const { value, setOff, setOn } = useToggle(true)
  return (
    <>
      <MessageBar intent="warn" onDismiss={setOff} visible={value}>
        I can be closed and reopened
      </MessageBar>
      {!value && (
        <ButtonOutline m="medium" onClick={setOn}>
          Show MessageBar
        </ButtonOutline>
      )}
    </>
  )
}
```
