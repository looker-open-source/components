## Button Styles

### Button

Use a default button for the most frequently used action or most important action on a page. Default buttons should only appear once on a page. They are sometimes accompanied by a secondary button.

```jsx
<Button>Default Button</Button>
```

### ButtonOutline

Use an outline button alongside a default button to provide alternative actions on a page. Outline buttons should only appear alongside default buttons for secondary actions. Use no more than two secondary buttons on a page.

```jsx
<ButtonOutline>Outline</ButtonOutline>
```

### ButtonTransparent

Use a transparent button as a tertiary action on a screen, they are often used as a Cancel button on a form. Typically it performs the opposite action of a default button

```jsx
<ButtonTransparent>Transparent</ButtonTransparent>
```

## Colors

Buttons come in three color variants: `key`, `critical`, and `neutral`.

### Key

Key is the branded default color, which is the default setting.

```jsx
<Space>
  <Button color="key">Key</Button>
  <ButtonOutline color="key">Key Outline</ButtonOutline>
  <ButtonTransparent color="key">Key Transparent</ButtonTransparent>
</Space>
```

### Critical

Critical Buttons are to be used in situations where you need to convey some very important, potentially irreversible consequence of pressing this button.

```jsx
<Space>
  <Button color="critical">Critical</Button>
  <ButtonOutline color="critical">Critical Outline</ButtonOutline>
  <ButtonTransparent color="critical">Critical Transparent</ButtonTransparent>
</Space>
```

### Neutral

Neutral buttons are meant to fade into the background. These are usually used for secondary, nonessential, actions.

```jsx
<Space>
  <Button color="neutral">Neutral</Button>
  <ButtonOutline color="neutral">Neutral Outline</ButtonOutline>
  <ButtonTransparent color="neutral">Neutral Transparent</ButtonTransparent>
</Space>
```

## Size

Use the size property on a `<Button />` to modify the size the button rendered. You can combine it with the `mode` property to get the correct style and size of button you need.

```jsx
<SpaceVertical>
  <Space>
    <Button size="xsmall">Xsmall Button</Button>
    <Button size="small">Small Button</Button>
    <Button>Medium (default) Button</Button>
    <Button size="large">Large Button</Button>
  </Space>
  <Divider />
  <Space>
    <ButtonOutline size="xsmall">Xsmall Button</ButtonOutline>
    <ButtonOutline size="small">Small Button</ButtonOutline>
    <ButtonOutline>Medium (default) Button</ButtonOutline>
    <ButtonOutline size="large">Large Button</ButtonOutline>
  </Space>
  <Divider />
  <Space>
    <ButtonTransparent size="xsmall">Xsmall Button</ButtonTransparent>
    <ButtonTransparent size="small">Small Button</ButtonTransparent>
    <ButtonTransparent>Medium (default) Button</ButtonTransparent>
    <ButtonTransparent size="large">Large Button</ButtonTransparent>
  </Space>
</SpaceVertical>
```

## Full Width

Use the `fullWidth` property on a `<Button />` and it will set its width to 100%.

```jsx
<SpaceVertical>
  <Button fullWidth>Full Width</Button>
  <ButtonOutline fullWidth>Full Width</ButtonOutline>
  <ButtonTransparent fullWidth>Full Width</ButtonTransparent>
</SpaceVertical>
```

## Disabled

Use a disabled button to indicate to the user what action will be possible on a page once a prerequisite action is taken. Disabled buttons do not respond to user interaction.

```jsx
<Button disabled>I am disabled</Button>
```

## Button with an Icon

Buttons can have an icon before or after their content, using the `iconBefore` and `iconAfter` property. Each property accepts the [name of an icon](/#!/Icons)

```jsx
<Space>
  <Button iconBefore="CircleAdd">Add To Cart</Button>
  <Button iconAfter="Trash" color="critical">
    Delete
  </Button>
</Space>
```

## Additional Action Considerations

When a call to action requires additional steps, include an ellipsis at the end of the button text as a visual clue. This helps reinforce that there are additional steps that can take place before the action takes effect.

```jsx
<Space>
  <Button color="critical">Move to Trash...</Button>
  <Button>Print...</Button>
</Space>
```

## Extending Button

Sometimes you may want to extend the Button defaults to create a specific styling effect. That should be straightforward by wrapped a StyledComponent in another `styled` call to extend it like so:

<RoundBtn />

```jsx static
const RoundButton = styled(Button)`
  border-radius: 2rem;
`
```
