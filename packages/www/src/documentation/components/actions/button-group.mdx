## ButtonGroup

`<ButtonGroup />` behaves as a checkbox group. Use this when the user may select zero or more of a small set of options.

### Controlled

Control externally via `value` and `onChange` props. On individual `ButtonItem`, `value` is optional and will fall back to the text if absent.

```js
;() => {
  const [value, setValue] = useState(['CA'])
  return (
    <>
      <ButtonGroup value={value} onChange={setValue}>
        <ButtonItem value="CA">California</ButtonItem>
        <ButtonItem value="AK">Alaska</ButtonItem>
        <ButtonItem value="UT">Utah</ButtonItem>
      </ButtonGroup>
      <Text pl="small">Current selection: {value.join(', ')}</Text>
    </>
  )
}
```

### Uncontrolled

Use uncontrolled when you want to get the value by inspecting the underlying checkbox inputs – `name` prop is recommended.

```jsx
<ButtonGroup name="numbers">
  <ButtonItem selected>Three</ButtonItem>
  <ButtonItem>Four</ButtonItem>
</ButtonGroup>
```

## ButtonToggle

`<ButtonToggle />` behaves as radio button group. Use this when the user may select only one of a small set of options.

### Controlled

Control externally via `value` and `onChange` props. On individual `ButtonItem`, `value` is optional and will fall back to the text if absent.

```jsx
;() => {
  const [value, setValue] = useState('TypeScript')
  return (
    <>
      <ButtonToggle value={value} onChange={setValue}>
        <ButtonItem>Ruby</ButtonItem>
        <ButtonItem>TypeScript</ButtonItem>
        <ButtonItem>Python</ButtonItem>
      </ButtonToggle>
      <Text pl="small">Current selection: {value}</Text>
    </>
  )
}
```

### Uncontrolled

Use uncontrolled when you want to get the value by inspecting the underlying radio button inputs – `name` prop is recommended.

```jsx
<ButtonToggle name="fruits">
  <ButtonItem selected>Apples</ButtonItem>
  <ButtonItem>Oranges</ButtonItem>
  <ButtonItem>Bananas</ButtonItem>
</ButtonToggle>
```
