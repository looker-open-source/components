Calendar

Navigation Callbacks

We support four different callbacks to hook into various navigation events:

  • onNextClick: the user clicked the right arrow
  • onPrevClick: the user clicked the right arrow
  • onNowClick: the user clicked the month label to return to the current month
  • onMonthChange: the user navigated to a new month (could be through any mouse or keyboard interaction)

;() => {
  const onNextClick = date =>
    alert(`Clicked "Next" (${date.getMonth() + 1}/${date.getFullYear()})`)
  const onNowClick = date =>
    alert(`Clicked "Now" (${date.getMonth() + 1}/${date.getFullYear()})`)
  const onPrevClick = date =>
    alert(`Clicked "Prev" (${date.getMonth() + 1}/${date.getFullYear()})`)
  const onMonthChange = date =>
    alert(`Month Changed (${date.getMonth() + 1}/${date.getFullYear()})`)
  return (
    <Calendar
      onNextClick={onNextClick}
      onNowClick={onNowClick}
      onPrevClick={onPrevClick}
      onMonthChange={onMonthChange}
    />
  )
}

I18n

Install the date-fns package and import the necessary locale.

import { Calendar } from '@looker/components-date'
import ko from 'date-fns/locale/ko'
export const LocalizedCalendar = () => {
  return <Calendar locale={ko} />
}

firstDayOfWeek

The calendar will use the locale's default to determine which day the week starts on. Use the firstDayOfWeek prop to override this 0 for Sunday, 1 for Monday, etc.
